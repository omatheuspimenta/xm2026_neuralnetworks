História das Redes Neurais

A ideia de rede neural não é algo novo, foi apresentada em 1943 no trabalho de McCulloch e Pitts (modelo computacional de um neurônio). Mas por que somente recentemente elas ganharam força? Além disso, o que é uma rede neural?
Diversas definições já foram propostas:

  • Operacionalmente podemos defini-la como: “Sistema complexo que compreende nós e links representados por neurônios e suas conexões” (WARREN et al., 2013)
  • Em uma visão mais geral pode ser definida como: “Sistema de processamento de informações composto por um grande número de elementos simples que são interligados diretamente e que cooperam para realizar processamento distribuído paralelo a fim de resolver uma tarefa computacional desejada” (MACUKOW, 2016).

Quais problemas podem ser resolvidos com redes neurais ?

Quase qualquer problema hoje pode ser resolvido utilizando redes neurais, isso não significa que seja prático ou mesmo a melhor solução. Mas em geral temos problemas envolvendo:

  • Classificação
  • Regressão
  • Detecção de Padrões
  • Redução de Dimensionalidade
  • Segmentação de Imagens
  • Geração de Dados sintéticos

Formalizando o Problema

Aprendizado supervisionado

No aprendizado supervisionado nos temos os dados definido por:

\[ \mathcal{D} = \{(x_i,y_i)\}^{N}_{i=1} \]

O objetivo é aprender uma função \(f(x;\theta) \approx y\) na qual \(\theta\) são os parâmetros do modelo

Problema de Otimização

Nesse caso quando treinamos o modelo, estamos resolvendo um problema de otimização. A ideia é minimizar o risco que é dado por:

\[ R(\theta) = \mathbb{E}_{(x,y)\sim P}[\mathcal{L}(y, f(x_i;\theta))] \]

Contudo não temos acesso a verdadeira distribuição, por causa disso o que realmente fazemos é minimizar o risco empírico (ou seja com os nossos dados)

\[ \hat{R}(\theta) = \frac{1}{N}\sum_{i=1}^n \mathcal{L}(y_i, f(x_i;\theta)) \]

  • \(\mathcal{L} = \text{função de perda}\), ela é definada para verificar a distância entre \(y_i\) e \(f(x;\theta)\).

Como resolver esse problema de otimização ?

A partir do problema existem diversas abordagens para tentar encontrar a solução. Diversos deles vocês já devem ter vistos como:

  • Regressão Linear
  • Regressão Logística
  • Support Vector Machines
  • Redes Neurais

Perceptron

Perceptron - Neurônio

Perceptron foi originalmente proposto como uma abstração matemática de um neurônio biológico.

Perceptron - Clássico

Perceptron foi criado como um classificador binário para resolver o problema do aprendizado supervisionado. Dado um vetor de entrada \(x\), o modelo define uma função discriminante linear:

\[ g(x) = w^Tx + b \]

Em que \(w\) é o vetor de pesos e \(b\) bias. A regra de decisão induz um classificador binário:

\[ \hat{y} = sign(w^Tx + b), \qquad \hat{y} \in \{-1, +1\} \]

Interpretação Geométrica

O conjunto de pontos que satisfaz:

\[ w^Tx + b = 0 \]

define um hiperplano de decisão em \(\mathbb{R}^d\), particiona o espaço em dois semi-espaços:

  • \(g(x) > 0\) então o ponto está de um lado do hiperplano e podemos definilo como a classe (+1)
  • \(g(x) < 0\) está no outro lado e portanto classifcado como outra classe (-1).

Nesse caso o vetor peso \(w\) é normal ao hiperplano e controla a sua orientação, enquanto \(b\) desloca a sua posição.

Exemplo

Em um problema em que temos duas classes linearmaente separáveis com apenas 2 features, ou seja \(x \in \mathbb{R}^2\), o hiperplano se reduz a uma reta \(w_1x_1 + w_2x_2 + b = 0\).

Formulação como Problema de Otimização

O Perceptron pode ser visto como um algoritmo que busca \((w,b)\) tal que:

\[ y_i(w^Tx_i + b) > 0, \forall i \]

Quando isso não ocorre atualiza-se de forma iterativa para encontrar o melhor peso:

\[ w \leftarrow w + \eta y_i x_i \]

\[ b \leftarrow b + \eta y_i \]

Visão Moderna

O Perceptron pode ser interpretado como uma unidade computacional composta por:

  • 1° Combinação Linear: \(z = w^Tx + b\)
  • 2° Função de ativação: \(\hat{y} = \phi(z)\)

No perceptron clássico \(\phi(z) = sign(z)\), é substituido por funções de ativação mais modernas como ReLU, sigmoid, tanh, entre outras que veremos mais a frente.

Perceptron - Implementação

Antes de implementar um perceptron do zero em Python, vamos demonstrar, passo a passo, como ocorre a atualização dos pesos. Para isso, utilizaremos o problema lógico “OR”, considerando o seguinte conjunto de dados:

  • \(\mathcal{D} = {([0,1], 1), ([1,1], 1), ([1,0], 1), ([0,0], -1)}\)
  • Como o problema possui duas features, o objetivo é encontrar os parâmetros \([w_1, w_2, b]\).

Inicialmente, adotaremos os valores:

  • Pesos e viés iniciais: \([w_1, w_2, b] = [0,0,0]\)
  • Taxa de aprendizado: \(\eta = 1\)

Processo

  • 1ª interação: \(\hat{y}_i = sign(w^Tx_i + b) \Rightarrow sign([0,1]^T[0,0] + 0) = 0\). Como a predição está incorreta, atualizamos: \(w \leftarrow [0,0] + (1)[0,1] = [0,1]\) e \(b \leftarrow 0 + 1 = 1\).
  • 2ª interação: \(sign([1,1]^T[0,1] + 1) = 1\). Predição correta, não há atualização dos parâmetros.
  • 3ª interação: \(sign([1,0]^T[0,1] + 1) = 1\). Predição correta.
  • 4ª interação: \(sign([0,0]^T[0,1] + 1) = 1\). Predição incorreta, então: \(w \leftarrow [0,1] + (-1)[0,0] = [0,1]\) e \(b \leftarrow 1 - 1 = 0\).

Processo

  • 5ª interação: As predições permanecem corretas até \(sign([1,0]^T[0,1] + 0) = 0\). Como a predição está incorreta: \(w \leftarrow [0,1] + (1)[1,0] = [1,1]\) e \(b \leftarrow 0 + 1 = 1\).
  • 6ª interação: As predições seguem corretas até \(sign([0,0]^T[1,1] + 1) = 1\). Como a predição está incorreta: \(w \leftarrow [1,1] + (-1)[0,0] = [1,1]\) e \(b \leftarrow 1 - 1 = 0\).
  • 7ª interação: Novamente, as predições permanecem corretas até \(sign([0,0]^T[1,1] + 0) = 0\). Como a predição está incorreta: \(w \leftarrow [1,1] + (-1)[0,0] = [1,1]\) e \(b \leftarrow 0 - 1 = -1\).

Com isso, todas as amostras são classificadas corretamente. Portanto, os parâmetros finais do modelo são \(w = [1,1]\) e \(b = -1\).

Um único perceptron tem capacidade de resolver apenas problemas linearmente separáveis. Minsky & Papert provaram (Perceptrons, 1969) que existem limitações sobre o que as redes Perceptron são capazes de representar utilizando o problema clássico XOR:

Perceptron - Solução

Com a evolução das pesquisas em redes neurais e, principalmente, o avanço da tecnologia, essas limitações foram superadas através dos seguintes fatores:

  • Introdução das Redes Neurais Multicamadas (MLP - Multilayer Perceptron) que utilizam múltiplos neurônios organizados em várias camadas.
  • Introdução de funções de ativação mais complexas.
  • Novo algoritmo (backpropagation) para atualizar os parâmetros, permitindo atualização eficiente dos pesos, e possibilitando o treinamento de redes mais profundas.

Redes Neurais

Como podemos entender redes neurais ?

As redes neurais podem ser interpretadas como composições de funções capazes de modelar relações não lineares complexas.

\[ f(x) = f_L \circ f_{L-1} \circ \dots \circ f_1(x) \]

Até o momento, vimos uma possível função para cada camada:

\[ f_i(x) = \phi(w^T x + b) \]

Como podemos observar, essa mesma estrutura pode ser repetida diversas vezes. Cada uma dessas estruturas é chamada de camada; assim, redes neurais são formadas pela composição de múltiplas camadas.

Forward Pass

O foward pass é o cálculo da saída ou seja \(\hat{y} = f(x;\theta)\).

Função de Perda

A função de perda (loss function) é um componente essencial em redes neurais, pois define uma medida da diferença entre a previsão do modelo e o valor esperado. Formalmente:

\[ \mathcal{L}(y, \hat{y}) \]

A escolha da função de perda depende do problema a ser resolvido, em alguns casos, é até necessário definir uma função personalizada. A seguir, veremos as principais funções utilizadas na maioria dos cenários:

  • Erro Quadrático Médio (Mean Squared Error — MSE)
  • Binary Cross-Entropy e sua generalização, Cross-Entropy

Artigo interessante: https://arxiv.org/pdf/2307.02694

MSE

O Mean Squared Error (MSE) é utilizado principalmente em problemas de regressão, nos quais desejamos prever valores contínuos.

\[ \frac{1}{N} \sum (y - \hat{y})^2 \]

Binary Cross-Entropy

Essa função de perda é utilizada em problemas de classificação binária e está relacionada ao conceito de entropia de Shannon.

\[ \mathcal{L} = -[y \log(\hat{y}) + (1-y)\log(1-\hat{y})] \]

Cross-Entropy

A Cross-Entropy é uma generalização da Binary Cross-Entropy, utilizada em problemas de classificação multiclasse.

\[ \mathcal{L} = -\sum_i y_i \log(\hat{y}_i) \]

Backpropagation

Nosso objetivo é minimizar o risco empírico. Para isso, utilizamos o algoritmo de backpropagation, responsável por calcular o gradiente (\(\nabla\)) da função de custo. Em essência, o algoritmo calcula a derivada parcial da função de perda em relação a cada parâmetro do modelo, propagando o erro da última camada até a primeira (“de trás para frente”) por meio da regra da cadeia.

\[ \mathcal{L}(f(x), y) \rightarrow \frac{\partial \mathcal{L}(f(x), y)}{\partial w_i} \]

Pela regra da cadeia:

\[ \frac{\partial y}{\partial x} = \frac{\partial y}{\partial u} \frac{\partial u}{\partial x} \]

Atualização dos Pesos

Esse algoritmo é conhecido como Gradient Descent. Com ele, os parâmetros da rede neural podem ser atualizados da seguinte forma:

\[ \theta_{t+1} = \theta_t - \eta \nabla_\theta \mathcal{L} \]

Na prática, não é necessário calcular as derivadas manualmente, pois bibliotecas como PyTorch realizam o cálculo dos gradientes automaticamente. Ainda assim, compreender esse processo é fundamental para entender o treinamento de redes neurais.

Exemplo

Vamos aplicar esse processo em um exemplo simples (sem utilizar a regra da cadeia). Considere a função: \(f(x,y)=x^2+y^2\) Desejamos encontrar seu ponto de mínimo a partir dos valores iniciais \(x=4\) e \(y=2\). O gradiente da função é dado por:

\[\nabla f(x,y)=\left(\frac{\partial f}{\partial x},\frac{\partial f}{\partial y}\right)=(2x,2y)\]

  • 1ª interação: \(x^{(0)}=4\) e \(y^{(0)}=2\), então \(\nabla f(4,2)=(8,4)\). Utilizando \(\eta = 0.1\): \((4,2) - 0.1(8,4) = (3.2,1.6)\)

Observe que o valor da função diminui após a atualização:\(f(4,2)=16+4=20 \qquad f(3.2,1.6)=10.24+2.56=12.8\)

Exemplo

  • 2ª interação: \(x^{(1)}=3.2\) e \(y^{(1)}=1.6\), então \(\nabla f(3.2,1.6)=(6.4,3.2)\). Utilizando novamente \(\eta = 0.1\):

\[ (3.2,1.6) - 0.1(6.4,3.2) = (2.56,1.28) \]

Observe que os valores continuam se aproximando de \((0,0)\), que corresponde ao ponto de mínimo da função. Repetindo esse processo iterativamente, convergimos para o mínimo.

Estrutura do Modelo

Os principais elementos que definem uma rede neural são:

  • Camadas
  • Funções de Ativação
  • Função de Perda
  • Otimizador

Até agora vimos quais são as funções de perda, camada de perceptrons e também vimos algumas funções de ativação (sign e tanh). Nos próximos slides veremos outras camadas, outras funções de ativação e vamos entender um dos otimizadores modernos, o ADAM.

Funções de Ativação

As funções de ativação possuem um papel central nas redes neurais, pois são responsáveis por introduzir não linearidade ao modelo, elemento essencial para aprender padrões complexos.

Atualmente, existem diversas funções de ativação, cada uma com características, vantagens e limitações específicas. Entre as principais, destacam-se:

  • ReLU (Rectified Linear Unit), Leaky ReLU, RReLU e PReLU
  • ELU e SELU
  • Sigmoid, Tanh e Softmax
  • Funções mais modernas: GELU, Swish, Mish e APALU
  • Funções oscilatórias mais recentes: GCU, DSU, NCU, SQU e SSU

A seguir, veremos algumas delas com mais detalhes.

ReLU

A ReLU é atualmente a função de ativação mais utilizada em redes neurais, principalmente devido à sua simplicidade e eficiência computacional.

\[ReLU(x)=\max(0,x)\]

  • Vantagens: reduz o problema do Vanishing Gradient e possui baixo custo computacional.

  • Desvantagens: produz gradiente apenas para valores positivos. Para entradas negativas, a saída é zero, o que pode levar ao problema conhecido como Dying ReLU.

    Artigos interessantes: https://arxiv.org/pdf/1903.06733, https://arxiv.org/abs/2505.22074

Leaky ReLU

A Leaky ReLU foi proposta para reduzir o problema dos neurônios mortos (Dying ReLU). Para isso, a função adiciona uma pequena inclinação para valores negativos, controlada pelo parâmetro \(\alpha\).

\[Leaky\ ReLU(x)=\max(\alpha x,x)\]

  • Vantagem: mantém gradientes diferentes de zero para entradas negativas.
  • Desvantagem: exige a definição manual do parâmetro \(\alpha\), motivando o surgimento de variantes como RReLU e PReLU.

Sigmoid

A Sigmoid foi, durante muito tempo, a principal função de ativação utilizada em redes neurais. No entanto, ela apresenta algumas limitações, sendo a principal o Vanishing Gradient Problem, causado pela saturação para valores muito positivos ou negativos.

\[\sigma(x)=\frac{1}{1+e^{-x}}\]

Atualmente, a Sigmoid raramente é utilizada em camadas intermediárias. Ainda assim, ela continua sendo muito importante em problemas de classificação binária, pois sua saída varia entre \(0\) e \(1\), permitindo interpretar a saída da rede como uma probabilidade.

Softmax

A Softmax segue a mesma ideia da Sigmoid: transformar a saída da rede em probabilidades. Entretanto, ela é utilizada em problemas de classificação multiclasse.

\[Softmax(x_i)=\frac{e^{x_i}}{\sum_j e^{x_j}}\]

Exemplo: suponha que desejamos classificar uma imagem entre cachorro, gato e coelho. Se a saída da rede for: Cachorro: \(1.75\), Gato: \(3.90\) e Coelho: \(4.35\), a aplicação da Softmax resultaria em: obtemos as probabilidades: Cachorro: \(0.04\), Gato: \(0.37\) e Coelho: \(0.59\)

Assim, o modelo classificaria a imagem como coelho, por possuir a maior probabilidade.

GELU

Funções de ativação mais modernas, como a GELU, possuem maior capacidade de modelar padrões complexos, geralmente ao custo de uma maior complexidade computacional.

A GELU é uma função suave, isto é, derivável em todos os pontos, e atualmente é amplamente utilizada em modelos baseados em Transformers.

\[GELU(x)=x\Phi(x)\approx0.5x\left(1+\tanh\left(\sqrt{\frac{2}{\pi}}\left(x+0.044715x^3\right)\right)\right)\]

APALU

A Adaptive Piecewise Approximated Activation Linear Unit (APALU) é uma função de ativação paramétrica, na qual a própria rede aprende os parâmetros da função durante o treinamento.

Por ser altamente adaptável e flexível, a APALU pode apresentar bom desempenho em problemas com padrões complexos.

\[APALU(x)=\begin{cases}a\left(x+x\left(\frac{1}{1+e^{-1.702x}}\right)\right),&x\geq0\\b(e^x-1),&x<0\end{cases}\]

GCU

A Growing Cosine Unit (GCU) foi uma das primeiras funções de ativação a introduzir comportamentos oscilatórios em redes neurais.

Sua principal contribuição foi mostrar que é possível resolver o problema XOR utilizando apenas um único neurônio. Apesar disso, suas propriedades e aplicações práticas ainda foram pouco exploradas.

\[GCU(x)=x\cos(x)\]

Camadas

Um modelo é basicamente feito por várias camadas, cada uma tendo uma função específica no modelo. As que vamos ver/vimos hoje são:

  • Dense: Camada mais comum de redes neurais. Composta por vários perceptrons, na qual cada perceptron está totalmente conectado a camada anterior.
  • Dropout: Camada que tem como objetivo evitar overfitting.
  • Batch Normalization: Estabiliza o treinamento e melhora a convergência.

Dropout

O que ele faz é desativar perceptrons aleatoriamente durante o treinamento, isso ajuda a reduzir o overfitting do modelo.

Obs: No teste essa camada é desativada e todos os perceptrons são utilizados

Batch Normalization

Principal objetivo dessa camada é estabilizar o treinamento das redes neurais normalizando os dados para ter media \(0\) e variância \(1\).

Batch Normalization

Quando não há normalização, algumas variáveis podem ter escalas muito diferentes, o que leva a gradientes desbalanceados. Com isso, uma variável pode ter uma magnitude muito maior no gradiente, enquanto outra tem magnitude muito pequena. Esse desbalanceamento pode prejudicar o desempenho dos algoritmos de otimização.

Tipos de Normalização

Existem diferentes estratégias de normalização utilizadas em redes neurais:

  • Batch Normalization: normaliza as ativações utilizando estatísticas calculadas ao longo do mini-batch.
  • Layer Normalization: realiza a normalização considerando todas as ativações de uma mesma camada para cada amostra.
  • Instance Normalization: normaliza cada amostra individualmente, sem depender das demais observações do batch.
  • Group Normalization: divide os canais em grupos e realiza a normalização separadamente em cada grupo.

Otimizadores

Embora o SGD (Stochastic Gradient Descent) apresente bons resultados, atualmente são utilizados, principalmente, otimizadores mais modernos, em sua maioria variantes do Adaptive Moment Estimation - ADAM.

Para entender essa evolução, é importante observar as principais limitações do SGD:

  • Utiliza uma taxa de aprendizado fixa.
  • Pode apresentar convergência lenta em funções com muitos parâmetros.

ADAM

O ADAM utiliza o conceito de Momentum, inspirado na física. Em vez de atualizar os parâmetros apenas com base no gradiente atual, o método considera também a direção acumulada dos gradientes anteriores, acelerando a convergência.

No ADAM, dois hiperparâmetros principais controlam esse processo: \(\beta_1\) e \(\beta_2\).

  • \(\beta_1\): controla a média exponencial dos gradientes passados (momento de primeira ordem).
  • \(\beta_2\): controla a média exponencial dos quadrados dos gradientes passados (momento de segunda ordem).

ADAM

As atualizações dos parâmetros seguem a fórmula:

\[ m_t = \beta_1 \cdot m_{t-1} + (1 - \beta_1) \cdot g_t \]

\[ v_t = \beta_2 \cdot v_{t-1} + (1 - \beta_2) \cdot g_t^2 \]

Onde: \(m_t\) é o momento de primeira ordem, \(v_t\) é o momento de segunda ordem e \(g_t\) é o gradiente no tempo \(t\).

ADAM

Além disso, o ADAM aplica uma correção para compensar as estimativas iniciais de \(m_t\) e \(v_t\):

\[ \hat{m}_t = \frac{m_t}{1 - \beta_1^t}, \quad \hat{v}_t = \frac{v_t}{1 - \beta_2^t} \]

Os parâmetros são atualizados da seguinte forma:

\[ \theta_t = \theta_{t-1} - \frac{\eta}{\sqrt{\hat{v}_t} + \epsilon} \cdot \hat{m}_t \]

ADAMW

Uma das variantes mais populares do ADAM é o ADAMW, que introduz o conceito de weight decay para regularização dos parâmetros do modelo.

A atualização dos parâmetros passa a ser definida por:

\[\theta_t=\theta_{t-1}-\frac{\eta}{\sqrt{\hat{v}_t}+\epsilon}\cdot\hat{m}t+\eta\lambda\theta{t-1}\]

O parâmetro \(\lambda\) controla a intensidade do weight decay, ajudando a reduzir overfitting e aumentando a estabilidade do treinamento.
